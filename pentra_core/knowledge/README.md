# Pentra Knowledge Governance

This directory holds cross-service knowledge governance artifacts used by the
public-source-first learning program.

Current files:

- `source_registry.yaml`
- `provenance_contract.yaml`
- `corpus/`
- `ontology/`
- `cheatsheets/`

These files do not store the learned corpus itself.
The `corpus/` directory does.

They define:

- which sources Pentra is allowed to learn from
- which sources are authoritative for which purposes
- how benchmark truth must be version-pinned
- what provenance fields future normalized facts must carry
- how Phase 1 acquisition manifests and normalized source summaries are stored
- how Phase 2 Pentra-native ontology objects are stored
- how structured cheat-sheet intelligence is categorized for future capability packs
